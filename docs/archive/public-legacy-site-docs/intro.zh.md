# 项目介绍

本页说明 `superSpider` 是什么，以及当前公开文档应该怎么读。

---

## superSpider 是什么？

`superSpider` 是一套围绕主脑构建的本地长期自治执行系统。

它不是“个人助理型聊天产品”，也不是单纯的对话壳。这个项目面向的是更长链的执行场景：规划、记忆、知识、运行态、证据和操作员控制都需要留在同一套可见系统里。

当前正式外部执行体路径是 `Codex`。后续会继续扩展更多执行体，但不会让执行体变成第二个主脑，也不会让它们变成第二套系统真相。

---

## 这套系统的关键区别是什么？

- **主脑控制面**：规划、派发、监督、恢复统一留在正式运行时里。
- **正式记忆与知识**：长期上下文不是一次性 prompt 填充物。
- **证据优先执行**：运行事件、证据与回放沿同一条回写链返回。
- **可见 Runtime Center**：运行态不应该埋在 worker 和日志里。
- **受管外部执行体**：执行层可以演进，但控制面不被替换。

## 当前命名边界

- 对外项目名：`superSpider`
- 当前 Python 包 / CLI 名称：`copaw`

也就是说，对外品牌已经切到 `superSpider`，但安装和本地命令目前仍然使用 `copaw`。
