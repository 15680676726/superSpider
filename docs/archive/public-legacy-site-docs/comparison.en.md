# Spider Mesh Capability Overview

This page describes the current Spider Mesh product surface only. It no longer uses legacy project comparison framing.

| Capability Area | Current Capability | Main Entry |
| :-- | :-- | :-- |
| Front-door control | The only default interactive front-door is **Spider Mesh Execution Core**, with a control thread plus task threads. | `/chat` |
| Team execution | One control brain with multiple specialist execution seats, sharing the same fact base and task chain. | `/industry`, `/runtime-center/agents` |
| Capability intake | Supports install templates, the SkillHub store, MCP, local skills, browser runtime, and desktop runtime. | `/capability-market` |
| Governance | Uses the unified `auto / guarded / confirm` model, with approvals handled in Runtime Center. | `/runtime-center/governance` |
| Evidence and review | Tasks, evidence, decisions, predictions, and reviews remain traceable through the runtime surface. | `/runtime-center`, `/predictions`, `/reports` |
| Industry bootstrap | Supports draft generation, team compilation, recommendation packs, and post-bootstrap autonomous operation. | `/industry` |
| Delivery | Supports local service mode, the Web console, desktop installers, and Docker delivery. | `/docs/quickstart`, `/docs/desktop` |
