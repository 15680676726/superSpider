# 2026-04-12 Runtime Center Human Cockpit Design

## Why This Exists

当前 `Runtime Center` 的主驾驶舱混入了大量普通用户不该先看到的系统词和运维块：

- 总览
- 治理
- 恢复
- 自动化

这会让页面像后台，而不是面向普通人的 AI 工作汇报台。

本设计把主驾驶舱收口成一个普通人能直接看懂、能快速知道“谁在干活、做到哪了、哪里卡住了”的页面，同时保留旧系统管理能力，但把它后置隐藏，不从首页删能力。

## Goals

1. 让普通用户第一眼看到多 agent 的工作状态，而不是后台运维信息。
2. 让主脑和职业 agent 的内容都变成简洁、固定结构的人话汇报。
3. 让“需要你决定”的事项从日报正文中拆出来，进入独立审批入口。
4. 保留治理、恢复、自动化能力，但不让它们污染普通用户主视图。

## Non-Goals

- 本次不删除现有治理、恢复、自动化后端能力。
- 本次不重新定义底层治理/恢复/自动化接口。
- 本次不把聊天控制全系统的能力一起做完。

## Page Contract

驾驶舱首页改成单一主结构：

- 上方一排 `agent 卡片`
- 下方一个 `当前选中 agent 的内容区`
- 默认选中 `伙伴（主脑）`
- 点击哪个 agent 卡片，下方就切到哪个 agent 的内容

页面不再在首页同时平铺大量无关区块。

## Agent Card Strip

### Ordering

- `伙伴（主脑）` 固定第一张
- 其他 agent 按优先级排序：
  - `卡住 / 待处理` 在前
  - 正常推进在后

### Card Fields

每张卡片只保留轻量信息：

- 头像
- 名字
- 职业
- 当前状态
- 进度条
- 待处理提醒图标

其中：

- 进度只用简化进度条，不堆字
- `待你处理` 用小手图标提示
- 卡片上不显示详细负责内容

## Content Tabs

### Common Agents

普通职业 agent 的内容区固定为：

- `简介`
- `日报`
- `统计`

### Main Brain

主脑内容区固定为：

- `简介`
- `日报`
- `统计`
- `阶段总结`
- `审批`
- `系统管理`

## Summary Tab

### Agent Summary

普通 agent 的 `简介` 固定显示：

- 我是谁
- 职责
- 主要负责工作
- 当前重点
- 最新进展
- 当前卡点

### Main Brain Summary

主脑的 `简介` 固定显示：

- 我是谁
- 职责
- 主要负责工作
- 当前重点
- 当前推进
- 最新结果
- 当前卡点

说明：

- `职责` 与 `主要负责工作` 必须写清楚，不允许空泛
- 内容必须短，不允许长篇 AI 自述

## Daily Report Tab

日报页只做汇报，不承接审批。

### Display Rules

- 白天默认显示 `早报`
- 白天如果没有 `晚报`，则不显示 `晚报`
- 晚上默认显示 `晚报`
- 晚上 `早报` 默认收起
- 点击后下拉展开
- 展开后顺序固定：`早报在上，晚报在下`
- 没生成的那份日报不显示

### Morning Report Structure

`早报` 固定为：

- 今天要做什么
- 重点先做什么
- 风险提醒

### Evening Report Structure

`晚报` 固定为：

- 今天完成了什么
- 产出了什么结果
- 明天继续什么

## Statistics Tab

统计页先做轻量版：

- 默认 `近7天`
- 可切换 `近30天`

固定指标：

- 完成数量
- 完成率
- 质量评分

图表：

- 使用 `ECharts`
- 先做 `折线图`

## Stage Summary Tab

`阶段总结` 只给主脑，不给普通职业 agent。

规则：

- 它不是硬性的周报/月报
- 只有运行时长和结果积累足够时才生成
- 系统刚运行很短时不生成伪阶段总结

## Approval Tab

### Positioning

- `聊天` 是第一通知入口
- `审批` 是第二兜底入口

也就是：

- 系统需要用户拍板时，先在聊天里正式询问
- 如果用户没及时处理，再把事项挂到主脑的 `审批` tab

### Scope

本次 `审批` tab 先只做：

- `待处理`

不做已处理历史。

### Approval Card Fields

每条审批项固定显示：

- 事项
- 原因
- 推荐
- 风险
- 发起人
- 发起时间
- 操作

操作先保留：

- 同意
- 拒绝
- 去聊天处理

规则：

- `同意` 二次确认
- `拒绝` 二次确认

## System Management Tab

`治理 / 恢复 / 自动化` 不删除，但从普通用户主驾驶舱首页后置隐藏。

它们统一收进主脑最后一个 tab：`系统管理`

### Structure

`系统管理` 内使用二级菜单：

- 自动化
- 治理
- 恢复

默认打开：

- `自动化`

### Product Rule

- 这些内容不再出现在首页主视野
- 它们仍然保留原有系统控制能力
- 它们属于后置系统层，而不是普通用户日常汇报层

## Copy Rules

本页所有新文案遵守以下规则：

- 用普通中文，不用后台术语堆叠
- 块要少，句子要短
- 不要长段 AI 自说自话
- 不要把审批、汇报、系统管理混写到一块

## Implementation Direction

前端实现时应遵守以下收口原则：

1. 首页只保留 `agent 卡片 + 当前 agent 内容区`
2. 主脑默认选中
3. 旧的 `治理 / 恢复 / 自动化` 从主页面入口移除，后置到主脑 `系统管理`
4. `审批` 独立，不再塞进 `日报`
5. `日报`、`简介`、`统计` 使用固定结构，不给 AI 无限自由发挥

## Expected Result

改完后，普通用户进入驾驶舱时应直接理解：

- 谁在工作
- 谁卡住了
- 今天推进到哪了
- 哪些结果已经出来了
- 哪些事等自己拍板

而不是先看到系统后台和运维信息。
